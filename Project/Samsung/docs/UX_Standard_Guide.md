#### GNB (Global Navigation Bar)

- **위치**: 상단 고정, 전체 너비, width:100%
- **구성 (좌→우)**:
  left
  - 현재 화면의 제목 + 제목에 맞는 아이콘
  right
  - 현재 날짜 (YYYY.MM.DD 요일)
  - 매뉴얼 버튼 (메뉴얼가이드 리스트 드롭다운운)
  - 사용자 프로필 버튼 (이름 + 드롭다운 아이콘)
- **높이**: 60px
- **배경색**: 흰색, 하단 border (`border-bottom: 1px solid #C9D0D7`)

## 2. 필터 패턴 (Filter Panel)

> 디자인 참조: `designs/samsung_raw/필터.png`, `designs/samsung_raw/Filter/Filters Set Modal.png`

### 2.1 필터 유형 선택 기준

| 조건 | 패턴 | 설명 |
|------|------|------|
| 검색 조건 **3개 이하** | 인라인 검색 | 테이블 상단에 직접 노출 (기존 방식) |
| 검색 조건 **4개 이상** | 필터 모달 | [Filter] 버튼 → 모달 패널에서 조건 입력 |

### 2.2 필터 모달 (Filter Modal)

**구조**

```
┌──────────────────────────────────────────────┐
│ ▼ Filter                                      │
├──────────────────────────────────────────────┤
│ 라벨1  ◉전체 ○옵션A ○옵션B │ 라벨4  [드롭다운 ▼]  │ 라벨7  [드롭다운 ▼]  │
│ 라벨2  [드롭다운 ▼]         │ 라벨5  [텍스트입력  ]  │ 라벨8  [텍스트입력  ]  │
│ 라벨3  [텍스트입력  ]        │ 라벨6  [텍스트입력  ]  │ 라벨9  [텍스트입력  ]  │
├──────────────────────────────────────────────┤
│ [필터 초기화]                    [취소] [필터 적용] │
└──────────────────────────────────────────────┘
```

**규칙**
- 3컬럼 그리드 배치 (라벨 + 입력 × 3열)
- 라디오 그룹: "전체" 옵션 포함, 선택된 항목은 `btn-primary` 스타일
- 하단 버튼: 좌측 `[필터 초기화]``outline-primary`, 우측 `[취소]``outline-secondary` + `[필터 적용]``primary`
- 필터 모달은 Bootstrap `modal-lg` 또는 `modal-xl` 사용

### 2.3 필터 태그 (Filter Chips)

필터 적용 후, 테이블 상단에 **적용된 필터를 태그(칩) 형태**로 표시한다.

```
┌──────────────────────────────────────────────────────────┐
│ ▼ Filter ② │ 국내/해외 : 전체 × │ OS : Window × │        ↺ 필터 초기화 │
└──────────────────────────────────────────────────────────┘
```

**규칙**
- `Filter` 버튼 옆에 적용된 필터 수를 **뱃지**로 표시 (예: `Filter ②`)
- 각 태그: `라벨 : 값` 형식 + 닫기(`×`) 버튼
- 태그 스타일: `badge bg-white border text-dark` + 닫기 아이콘
- 태그 `×` 클릭: 해당 필터만 제거 → 목록 자동 갱신
- `[필터 초기화]` 클릭: 모든 필터 제거 → 전체 목록 표시
- 우측 끝에 `[필터 초기화]` 링크 배치 (`↺` 아이콘 + `text-primary`)

### 2.4 인라인 검색 (기존 방식)

검색 조건이 3개 이하인 간단한 화면에서 사용한다.

- [검색], [초기화] 버튼은 검색 영역 우측 하단에 배치
- 기본값이 있는 필드는 페이지 로드 시 자동 설정 (예: 사용여부=사용, 기간=이번 달)

### 3.1 기본 규칙

- 페이징: 기본 20건 (10/20/50/100 선택 가능)
- 정렬: 컬럼 헤더 클릭 시 ASC/DESC 토글 (기본 정렬: 수정일시 DESC)
- 체크박스: 첫 번째 컬럼, 헤더에 전체 선택 체크박스
- 행 호버: 배경색 변경 (`table-hover`)
- 줄무늬: `table-striped`
- 컬럼 너비: 고정 너비 지정 (내용에 따라 테이블 깨지지 않도록)

### 3.2 셀 표시 규칙

| 데이터 유형 | 정렬 | 표시 형식 |
|-----------|------|---------|
| 텍스트 | 좌측 | 그대로 |
| 숫자 | 우측 | 천 단위 콤마 (1,234) |
| 퍼센트 | 우측 | N.N% (소수점 1자리) |
| 날짜 | 중앙 | YYYY-MM-DD |
| 일시 | 중앙 | YYYY-MM-DD HH:mm |
| 뱃지 | 중앙 | 아래 뱃지 규칙 참조 |
| 링크 (클릭 가능) | 좌측 | 밑줄 + 포인터 커서 |
| "-" (값 없음) | 중앙 | 회색 텍스트 |

### 3.3 셀 내 아이콘 표시

> 디자인 참조: `designs/samsung_raw/테이블 디자인 스타일.png`

테이블 셀에서 **텍스트와 함께 아이콘을 표시**하는 규칙이다.

**OS/시스템 아이콘**

| OS | 아이콘 | 표시 방식 |
|----|-------|---------|
| Windows | 🟦 (Windows 로고) | 아이콘 16px + 텍스트 |
| Linux | 🐧 (Tux 로고) | 아이콘 16px + 텍스트 |
| Oracle Solaris | ⭕ (Oracle 로고) | 아이콘 16px + 텍스트 |
| AIX | `AIX` (텍스트 뱃지) | `badge bg-light text-dark` |

- 아이콘 크기: 16px × 16px
- 아이콘과 텍스트 사이 간격: 8px (`me-2`)
- 아이콘은 `<img>` 태그 또는 PUI 아이콘 컴포넌트 사용
- 아이콘 파일 경로: `pui/img/icons/os/` (등록 필요)
- 아이콘이 없는 경우: 텍스트만 표시

### 3.4 트리 테이블 (S-02 등)

- 분류 노드: 접기/펼치기 아이콘 (▶/▼) + 들여쓰기 (depth × 24px)
- 분류 노드 행: 배경색 구분 (`bg-light`)
- 항목 행: 일반 행 스타일
- [전체 펼치기/접기] 버튼 제공

### 5.1 버튼 유형

| 유형 | Bootstrap 클래스 | 용도 |
|------|----------------|------|
| 주요 액션 | `btn btn-primary` | [저장], [검색], [제출], [확정] |
| 보조 액션 | `btn btn-outline-secondary` | [초기화], [목록], [취소] |
| 위험 액션 | `btn btn-danger` | [삭제] |
| 성공 액션 | `btn btn-success` | [활성화], [확인완료], [승인] |
| 경고 액션 | `btn btn-warning` | [비활성화], [반려], [재조치요청] |
| 정보 액션 | `btn btn-outline-primary` | [엑셀 다운로드], [복사] |

### 5.2 버튼 배치 규칙

**검색 영역**: [검색]`primary` + [초기화]`outline-secondary` → 우측 정렬

**목록 상단**: 좌측 [등록]`primary` + [삭제]`danger`, 우측 [엑셀 다운로드]`outline-primary`

**상세 하단**: [저장]`primary` + [삭제]`danger` + [목록]`outline-secondary` → 중앙 또는 우측 정렬

**상태 변경 버튼**: 현재 상태에 따라 동적 표시
- 운영상태=대기/비활성 → [활성화]`success` 표시
- 운영상태=활성 → [비활성화]`warning` 표시

### 5.3 버튼 크기

- 기본: `btn` (표준 크기)
- 테이블 내: `btn btn-sm` (소형)
- 모달 내: `btn` (표준)

![슬라이드 패널](../designs/design-system/Modal/Slide.png)

#### 슬라이드 패널 구조

```
┌────────────────────────────────────────────┐
│ ← 제목                                  × │
├────────────────────────────────────────────┤
│                                            │
│  섹션 1. 기본 정보                          │
│  ┌────┬────┬────┬────┐                    │
│  │라벨 │값   │라벨 │값   │                    │
│  └────┴────┴────┴────┘                    │
│                                            │
│  섹션 2. 상세 정보                          │
│  ┌──────────────────────┐                 │
│  │ 테이블 또는 폼 영역     │                 │
│  └──────────────────────┘                 │
│                                            │
├────────────────────────────────────────────┤
│                     [취소] [저장/수정/적용]   │
└────────────────────────────────────────────┘
```

#### 기본 규칙

| 항목 | 규칙 |
|------|------|
| 너비 | 기본 1000px (복잡한 내용), 800px (단순 폼) |
| 열림 방향 | 우측에서 좌측으로 슬라이드 |
| 배경 오버레이 | 반투명 검정 (`rgba(0,0,0,0.3)`) |
| 닫기 | 우측 상단 `×` 버튼 또는 배경 클릭 |
| 헤더 | `←` 뒤로가기 (2중 슬라이드 시) + 제목 + `×` 닫기 |
| 하단 버튼 | 우측 정렬, `[취소]``outline-secondary` + `[주요액션]``primary` |
| 스크롤 | 패널 내부 독립 스크롤 (본문 스크롤 잠금) |

#### 2중 슬라이드 패널

하나의 슬라이드 패널 위에 **두 번째 패널이 겹쳐서 열리는** 구조이다.

```
┌──── 1차 패널 (배경) ────┬──── 2차 패널 (전면) ────┐
│                        │ ← 제목              × │
│ (어둡게 비활성화)        │                       │
│                        │  입력 폼 / 상세 정보    │
│                        │                       │
│                        │          [취소] [저장]  │
└────────────────────────┴───────────────────────┘
```

- 2차 패널 닫기 → 1차 패널로 복귀 (`←` 뒤로가기 버튼)
- 1차 패널은 어둡게 비활성화 표시
- 최대 2단계까지만 허용 (3중 슬라이드 금지)

#### 사용 용도

| 용도 | 예시 화면 |
|------|---------|
| 상세 정보 조회/수정 | 자산 상세정보, 담당자 정보 수정 |
| 변경 이력 조회 | 담당자 변경 이력 |
| 예외/제외 신청 | 점검 예외 신청 (2중 슬라이드) |
| 하위 데이터 입력 | 점검 결과 입력, 조치 계획 입력 |

> 디자인 참조: `designs/samsung_raw/점검계획 관리.png` (상단 연두색 배너), `designs/samsung_raw/점검계획 /점검 대상별 관리.png` (상단 안내 영역)

화면 상단에 **정보성 안내 메시지를 인라인으로 표시**하는 패턴이다. 토스트와 달리 자동 소멸하지 않고 화면에 고정된다.

```
┌─────────────────────────────────────────────────────────────┐
│ ⓘ 안내 메시지 내용이 여기에 표시됩니다.                         × │
└─────────────────────────────────────────────────────────────┘
```

| 유형 | 배경색 | 테두리 | 아이콘 | 용도 |
|------|-------|--------|-------|------|
| 정보 | 연한 녹색 (`#e8f5e9`) | 좌측 4px 녹색 바 | `ⓘ` | 일반 안내, 연동 상태 안내 |
| 경고 | 연한 주황색 (`#fff3e0`) | 좌측 4px 주황색 바 | `⚠` | 주의 필요 사항 |
| 오류 | 연한 빨간색 (`#fce4ec`) | 좌측 4px 빨간색 바 | `⊘` | 오류 발생, 데이터 불일치 |

**규칙**
- 위치: 콘텐츠 영역 최상단 (화면 제목 아래, 요약 포틀릿 위)
- 닫기(`×`) 버튼: 우측 상단, 클릭 시 배너 숨김 (세션 내 유지)
- 텍스트: 14px, 좌측 아이콘 + 메시지
- 여러 줄 가능 (`●` 불릿으로 나열)
- 강조 텍스트: `font-weight: bold` 또는 `text-primary` 색상

## 9. 강조/경고 표시 표준

### 9.1 기한 초과

- 내부조치기한/감독기관조치기한 초과: 셀 텍스트 **빨간색** (`text-danger`)
- 종료일 임박 (D-3 이내): 셀 텍스트 **빨간색** + 볼드

### 9.2 프로그레스바

- 진행률/완료율/보고율: Bootstrap `progress-bar`
- 80% 이상: 녹색 (`bg-success`)
- 50~80%: 노란색 (`bg-warning`)
- 50% 미만: 빨간색 (`bg-danger`)

#### 분할 프로그레스바 (Stacked Progress Bar)

> 디자인 참조: `designs/samsung_raw/점검 결과 관리.png` (테이블 내 Pass/Fail 바)

테이블 셀 안에서 **복수 항목의 비율을 하나의 바에 분할 표시**하는 패턴이다.

```
┌──────────────────────────────────┐
│ ████████████████░░░░░░░░         │ Pass 388 / Fail 35
│  (녹색 bg-success)(빨간색 bg-danger) │
└──────────────────────────────────┘
```

- Bootstrap `progress-stacked` 사용
- 셀 내 배치 시 `height: 8px`, `border-radius: 4px`
- 바 우측 또는 하단에 수치 텍스트 표시 (예: `Pass 388 · Fail 35`)
- 색상: Pass=녹색(`bg-success`), Fail=빨간색(`bg-danger`), 미점검=회색(`bg-secondary`)

### 9.3 요약 카드 (대시보드)

- 카드 배경: 흰색 + 좌측 색상 바 (4px, 지표 유형별 색상)
- 숫자: 큰 폰트 (24px~32px)
- 라벨: 작은 폰트 (14px), 회색
- 변동: 전기 대비 증감 화살표 (▲녹색, ▼빨간색)

### 9.4 요약 포틀릿 (Summary Portlet)

> 디자인 참조: `designs/samsung_raw/요약포틀릿-점검계획관리.png`, `designs/samsung_raw/접힌 요약포틀릿.png`

목록 화면 상단에 **핵심 지표를 요약 표시**하는 영역이다. 대시보드의 요약 카드와 달리, 목록 화면에서 테이블 위에 배치된다.

#### 펼침 상태 (기본)

```
┌──────────────────────────────────────────────────────────────┐
│ ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐ │
│ │ 📋 전체 점검 계획  │ │ ⚙ 진행중인 점검   │ │ ⓘ 점검 오류 정보  │ ∧│
│ │                  │ │                  │ │                  │ │
│ │     1,420       │ │       45        │ │       12        │ │
│ │                  │ │                  │ │                  │ │
│ │ ● 전체 대상  610  │ │ ● 전체 대상  12   │ │ ● 전체 대상   3  │ │
│ │ ● 사업장   490  │ │ ● 사업장    9   │ │ ● 사업장    1  │ │
│ │ ● 개별 대상  320  │ │ ● 개별 대상  24   │ │ ● 개별 대상   8  │ │
│ └─────────────────┘ └─────────────────┘ └─────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

#### 접힌 상태

```
┌──────────────────────────────────────────────────────────────┐
│ 📋 전체 점검 계획 1,420 ●610 ●490 ●320 │ ⚙ 진행중인 점검 45 ●12 ●9 ●24 │ ⓘ 점검 오류 12 ●12 ●9 ●24 ∨│
└──────────────────────────────────────────────────────────────┘
```

#### 규칙

| 항목 | 규칙 |
|------|------|
| 배경 | 흰색 카드, `border-radius: 8px`, `box-shadow` 적용 |
| 카드 수 | 2~4개 (화면별 상이) |
| 주요 숫자 | 큰 폰트 (28px~36px), `font-weight: bold`, 색상 구분 (기본: 남색, 오류: 빨간색) |
| 세부 항목 | `●` 색상 도트 + 라벨 + 숫자, 작은 폰트 (14px) |
| 접기/펼치기 | 우측 `∧`(접기) / `∨`(펼치기) 아이콘, 클릭 토글 |
| 접힌 상태 | 한 줄로 축약 — 아이콘 + 라벨 + 주요 숫자 + 색상 도트 숫자 나열 |
| 기본 상태 | 펼침 |
| 위치 | 알림 배너(있을 경우) 아래, 필터 영역 위 |

## 10. 네비게이션 표준

### 10.1 빵부스러기 (Breadcrumb)

```
홈 > 상시 보안점검 > 항목 설정 목록 > 항목 설정 등록
```

- 마지막 항목은 현재 페이지 (볼드, 클릭 불가)
- 이전 항목은 클릭 시 해당 화면으로 이동

### 10.2 목록 ↔ 상세 이동

- 목록에서 항목명 클릭 → 상세/수정 화면으로 이동
- 상세에서 [목록] 버튼 → 이전 목록 화면으로 복귀 (검색 조건 유지)
- 브라우저 뒤로가기 지원 (History API)

### 10.3 탭 네비게이션

![Basic Tab](../designs/design-system/Tab/Basic-Tab.png)

- 한 화면 내 섹션 전환: 수평 탭 (`nav-tabs`)
- 탭 전환 시 데이터 유지 (DOM 숨김/표시 방식)

## 11. 접근성 표준

### 툴팁 (Tooltip)

![Tooltip](../designs/design-system/Etc/Tooltip.png)

### 11.1 기본 규칙

- 모든 입력 필드에 `<label>` 연결 (`for` 속성)
- 이미지/아이콘에 `alt` 또는 `aria-label` 제공
- 키보드 네비게이션: Tab 순서 논리적 배치
- 포커스 표시: 입력 필드 포커스 시 테두리 강조 (Bootstrap 기본)

### 11.2 색상 대비

- 텍스트 대비율: WCAG 2.1 AA 기준 (4.5:1 이상)
- 뱃지 텍스트: 배경 대비 가독성 보장 (예: `bg-warning`에 `text-dark`)

## 12. 도메인 특화 UX 규칙

### 12.1 상태 변경 확인

- 운영상태, 도출상태 등 **상태 변경 시 반드시 확인 다이얼로그** 표시
- 되돌릴 수 없는 변경: 빨간색 경고 메시지 추가

### 12.2 삭제 보호

- 참조 중인 데이터 삭제 시 차단 (409 Conflict) → 경고 메시지에 참조 건수 표시
- 논리 삭제 기본 (물리 삭제 없음)
- 삭제 확인 다이얼로그에 삭제 대상 정보 표시

### 12.3 임시저장/제출 패턴

- [임시저장]: 필수 검증 없이 즉시 저장 → 토스트 "임시저장되었습니다."
- [제출]: 필수 필드 검증 → 확인 다이얼로그 "제출하시겠습니까? 제출 후에는 수정이 제한됩니다." → 토스트 "제출되었습니다."
- [재제출]: 확인 다이얼로그 "재제출하시겠습니까? 담당자에게 재검토 알림이 발송됩니다."

### 12.4 제외처리/예외처리

- [제외처리]: 사유 입력 팝업 (사유 필수) → 확인 → 토스트
- [예외요청]: 체크박스 토글 → 예외의견 영역 동적 표시

### 12.5 기한 관리

- 기한 초과 건: 목록에서 빨간색 강조
- 기한 임박 건 (D-3 이내): 주황색 강조
- 기한 필드에 D-N 표시 (예: "2026-04-15 (D-5)")
