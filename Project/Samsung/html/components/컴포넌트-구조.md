# 컴포넌트 구조 정리

## 1. 개요

페이지 공통 영역(LNB, GNB)과 자산 상세 슬라이드는 **별도 HTML/JS 파일**로 분리해 두고,  
각 페이지에서는 **placeholder**만 두고 `load.js` 한 번만 로드하면 자동으로 채워지도록 되어 있습니다.

---

## 2. 폴더/파일 구성

```
Project/Samsung/html/
├── index.html              # 대시보드 (placeholder만 있음)
├── asset-status.html       # 자산 현황 (placeholder + asset-detail-placeholder)
├── home.html               # 메뉴 (대시보드/자산현황 링크)
├── style.css               # 공통 + 페이지 스타일
├── script.js               # 대시보드용 스크립트 (load.js가 동적 로드)
└── components/
    ├── load.js                  # ★ 진입점: LNB/GNB/자산테이블/자산상세 로드 후 script.js 로드
    ├── lnb.html                 # 좌측 사이드 메뉴 (LNB)
    ├── gnb.html                 # 상단 글로벌 네비 (GNB)
    ├── asset-table-section.html # 자산 현황 테이블 카드 (툴바 + 테이블 헤더 + 페이지네이션)
    ├── asset-table-tbody-rows.html # 자산 테이블 tbody 행 조각 (동적 주입용)
    ├── asset-detail-slide.html   # 자산 상세 1단/2단 슬라이드 마크업
    ├── asset-detail-slide.js    # 자산 상세 슬라이드 동작 (테이블 클릭 → 패널 열기 등)
    └── README.md                # LNB/GNB 사용법 요약
```

## 3. Placeholder → 컴포넌트 매핑

| Placeholder ID | 주입 파일 | 사용 페이지 | 역할 |
|----------------|-----------|------------|------|
| `#lnb-placeholder` | `components/lnb.html` | index, asset-status | 좌측 네비게이션 (메뉴, 접기) |
| `#gnb-placeholder` | `components/gnb.html` | index, asset-status | 상단 바 (제목, 매뉴얼, 프로필) |
| `#asset-table-section-placeholder` | `components/asset-table-section.html` + `asset-table-tbody-rows.html` | **asset-status만** | 테이블 툴바(전체 N건 + 버튼), 테이블, 페이지네이션 |
| `#asset-detail-placeholder` | `components/asset-detail-slide.html` + `asset-detail-slide.js` | **asset-status만** | 자산 상세 1단/2단 슬라이드 패널 |

- **index.html**: LNB + GNB만 사용 (asset-detail-placeholder 없음)
- **asset-status.html**: LNB + GNB + 자산 상세 슬라이드 전부 사용

## 4. 로드 순서 (load.js)

### 4.1 병렬 fetch (Promise.all)

서로 의존성이 없는 HTML은 **동시에** 요청해 렌더링 속도를 줄입니다.

- `lnb.html`, `gnb.html` — 항상 병렬 fetch
- `asset-table-section.html`, `asset-table-tbody-rows.html` — `#asset-table-section-placeholder`가 있을 때만 병렬 fetch
- `asset-detail-slide.html` — `#asset-detail-placeholder`가 있을 때만 fetch

특정 파일만 실패해도 **전체가 멈추지 않도록** 각 fetch는 실패 시 콘솔에 로그만 남기고 `null`을 반환합니다.

### 4.2 DOM 주입 (순서 고정)

fetch가 모두 끝난 뒤, 아래 순서로만 DOM에 주입합니다.

1. `#lnb-placeholder`, `#gnb-placeholder`에 **innerHTML** 주입
2. **GNB 제목** 설정: `<meta name="gnb-title" content="…">` 값으로 상단 제목 교체
3. **#asset-table-section-placeholder**가 있으면  
   - `asset-table-section.html` → placeholder 주입  
   - `asset-table-tbody-rows.html` 파싱 후 `#asset-table-tbody`에 주입  
   - `asset-table-ready` 커스텀 이벤트 dispatch (테이블 관련 스크립트 초기화용)
4. **#asset-detail-placeholder**가 있으면 `asset-detail-slide.html` → placeholder 주입

### 4.3 스크립트 실행 (DOM 조립 완료 후)

**모든 HTML 주입이 끝난 뒤**에만 스크립트를 로드합니다. (Race Condition 방지)

1. `#asset-detail-placeholder`가 있으면 **asset-detail-slide.js** 동적 `<script>` 로드 (await)
2. **script.js** 동적 `<script>` 로드 (await)

스크립트도 로드 실패 시 해당만 콘솔 로그하고, 나머지는 계속 진행합니다.

## 5. 페이지에서 쓰는 방법

### 공통 (대시보드 / 자산현황 공통)

- HTML에 **placeholder** 두기:
  - `#lnb-placeholder` (LNB 영역)
  - `#gnb-placeholder` (GNB 영역)
- 상단 제목은 `<head>` 안에:
  - `<meta name="gnb-title" content="페이지 제목">`
- 스크립트는 **load.js 하나만**:
  - `<script src="components/load.js"></script>`  
  (script.js는 load.js가 알아서 불러옴)

### 자산 현황에서만 추가

- 본문에 테이블 영역 placeholder:
  - `<section class="asset-table-section ..." id="asset-table-section-placeholder"></section>`  
  → **테이블 툴바**(전체 N건 + 서버 정보 수정/동기화/다운로드), **테이블**, **페이지네이션**이 주입됨.  
  - 전체 건수 갱신 시 `document.getElementById('asset-table-total-count').textContent = N` 사용.
- 본문 아래(예: `</main>` 바깥)에:
  - `<div id="asset-detail-placeholder"></div>`
- 이렇게 두면 **자산 테이블 섹션**과 **자산 상세 슬라이드** HTML/JS가 위 순서대로 주입·실행됨.

## 6. 컴포넌트별 역할 요약

| 컴포넌트 | 파일 | 역할 |
|----------|------|------|
| **로더** | `load.js` | Promise.all로 HTML 병렬 fetch → DOM 순차 주입 → **조립 완료 후** asset-detail-slide.js / script.js 순차 로드. 개별 실패 시 로그만 하고 나머지 진행. |
| **LNB** | `lnb.html` | 좌측 메뉴, 서브메뉴, 접기/펼치기 UI |
| **GNB** | `gnb.html` | 상단 바, 매뉴얼/프로필 팝오버 등 |
| **자산 테이블 섹션** | `asset-table-section.html` + `asset-table-tbody-rows.html` | 툴바(전체 N건, 버튼), thead + tbody, 행 개수 선택·페이지네이션. `#asset-table-total-count`로 건수 갱신 |
| **자산 상세 슬라이드** | `asset-detail-slide.html` + `asset-detail-slide.js` | 테이블 서버명 클릭 → 1단 패널, "자산 변경 이력" 클릭 → 2단 패널, 빈 곳 클릭 시 단계별 닫기 |

## 7. 참고

- **style.css**: LNB 접힘 등은 `.lnb-collapsed` 등으로 `#lnb-placeholder` 너비 제어.
- **자산 상세**: `#asset-detail-placeholder`는 레이아웃에서 빼고(위치/크기 0 등) 내부 슬라이드만 `position: fixed`로 표시하도록 CSS 처리됨.
- **테이블 초기화 타이밍**: 테이블이 비동기로 주입되므로, 행 개수·체크박스 등 테이블 기능은 `document.addEventListener('asset-table-ready', initAssetTable)` 로 **DOM 조립 후** 한 번만 실행하도록 되어 있음.

## 8. 폰트 스타일 규칙

> **공식 확장:** 동일 내용을 장·강제 규칙까지 포함해 **`Project/Samsung/docs/UX-STANDARD.md` §2 (Typography & Font Style Rules)** 에 정의한다.

모든 화면·컴포넌트는 **`style.css`에 정의된 폰트 스택**을 따른다. 페이지나 컴포넌트 전용 CSS에서 `font-family`를 임의로 바꾸지 않는다.

### 8.1 본문·UI 텍스트 (Pretendard)

- **CSS 변수:** `var(--font-pretendard)`  
  - 정의: `'Pretendard Variable', Pretendard, -apple-system, BlinkMacSystemFont, system-ui, sans-serif`
- **적용 범위:** `:root` 기본값, `body`, `#app` 및 그 하위의 버튼·카드·폼·테이블·드롭다운 등은 `style.css`에서 동일 스택으로 통일됨.
- **규칙:** 신규 마크업도 `#app` 내부에 두거나, 동일한 `font-family: var(--font-pretendard)` 패턴을 유지한다.

### 8.2 기본 타이포그래피 (글로벌)

- **기본 줄간격:** `line-height: 1.5` (`:root`)
- **기본 본문 색:** 페이지 배경·텍스트는 토큰과 함께 쓰는 것을 권장 (`--color-text-primary` 등, 아래 §9 참고).
- **렌더링:** `-webkit-font-smoothing: antialiased`, `-moz-osx-font-smoothing: grayscale` 유지.

### 8.3 아이콘 전용 (Material Symbols)

- 본문 폰트와 **별도**: `<span class="material-symbols-outlined">` 등은 Google Material Symbols 폰트를 사용한다.
- 아이콘 크기·`font-variation-settings`(FILL 등)는 화면별로 조정 가능하나, **아이콘 라이브러리 자체는 `UX-STANDARD.md` §9 Iconography** 를 따른다.

## 9. Design Tokens & Theme Constraints (디자인 토큰 및 CSS 규칙)

고객사·테마 확장을 위해 **브랜드·경계·배경·상태 색은 가능한 한 CSS 변수만** 사용한다. 임의 HEX를 컴포넌트 CSS에 남발하지 않는다. (부득이한 경우에도 기존 토큰과 의미가 겹치면 토큰 추가를 검토한다.)

### 9.1 필수 준수 (UX 표준과 동일 축)

| 토큰 | 용도 |
|------|------|
| `var(--color-primary)` | 메인 브랜드, 활성 탭, 주요 아이콘, 포커스 아웃라인 등 |
| `var(--color-primary-hover)` | Primary 호버(명도 낮춤) |
| `var(--color-primary-10)` 등 | 연한 브랜드 배경(선택 행, 뱃지 배경 등). 문서/코드에서 `--color-primary-light`와 동일 계열로 취급 가능 |
| `var(--color-border)`, `var(--color-border-light)` 등 | 기본·연한 테두리 (`style.css` `:root`의 border 별칭·Black 알파 계열) |

### 9.2 `style.css` `:root` 토큰 요약 (구현 기준)

**Primary**

- `--color-primary`, `--color-primary-hover`, `--color-primary-80` / `-60` / `-40` / `-20` / `-10` / `-5`, `--color-primary-border`, `--color-primary-bg`
- 별칭: `--color-primary-bg` → `--color-primary-10`

**Black / White**

- `--color-black-100` ~ `--color-black-4`, `--color-white-100`

**Background**

- `--color-bg-base`, `--color-bg-subtle`, `--color-bg-white`

**Text**

- `--color-text-primary`, `--color-text-secondary`, `--color-text-muted`

**Status (Figma 3단: 본색 / Light / White)**

- 본색: `--color-status-purple`, `indigo`, `blue`, `cyan`, `green`, `yellow`, `orange`, `red`
- Light: `--color-status-light-*` (파스텔 솔리드, 흰 글자 권장)
- White: `--color-status-white-*` (옅은 배경, 진한 글자 권장)
- 호환: `--color-status-indigo-10` 등 `*-10` → White 티어와 동일 용도

**레이아웃·데코**

- `--shadow-card`, `--shadow-popover`
- `--radius-card`, `--radius-button`, `--radius-popover`, `--radius-badge`
- 기타 별칭: `--color-border`, `--color-border-panel`, `--color-card-accent` 등 (`style.css` 상단 `:root` 블록 참고)

### 9.3 작성 규칙 (CSS)

1. **색상:** 새 스타일은 위 변수 우선. Bootstrap 유틸과 병용 시에도 **브랜드·강조색은 토큰**으로 맞춘다.
2. **모서리·그림자:** `border-radius`, `box-shadow`는 가능하면 `--radius-*`, `--shadow-*` 사용.
3. **테마 전환:** 고객사별로는 `:root` 또는 상위 스코프에서 변수 값만 교체하는 방식을 전제로 한다.
4. **충돌:** Bootstrap 등과 겹칠 때는 프로젝트 규칙에 맞게 선택자 구체화 또는 문서화된 `!important` 사용(기존 `style.css` 패턴 유지).

상세 UX 규칙은 **`Project/Samsung/docs/UX-STANDARD.md`** 를 함께 본다.

## 10. Iconography (아이콘 사용 규칙)
- **라이브러리 강제:** 프로젝트의 모든 아이콘은 오직 `Google Material Symbols (Outlined)`만 사용한다. 
- **금지 사항:** FontAwesome, Lucide, Heroicons 등 타 라이브러리 사용 및 SVG Path 직접 드로잉을 절대 금지한다.
- **알맞는 아이콘이 없을때:** 아이콘을 렌더링할 때는 알맞는 아이콘이 없을경우 최대한 비슷한 아이콘을 골라서 사용한다.
- **코드 작성 포맷:** 아이콘을 렌더링할 때는 반드시 아래의 HTML/컴포넌트 구조를 따른다.
  - `(예시) <span class="material-symbols-outlined">search</span>`

## 11. Company Logo Image (기업 로고 자동 배치 규칙)
- **로고 경로:** 기업 로고 이미지는 `/assets/logos/` 폴더 내에 위치한다.
- **파일명 매칭 (Naming Rule):** - AI는 화면에 기업 로고를 렌더링할 때, 전달받은 데이터의 `company_id` 또는 `영문 기업명`을 조합하여 이미지 경로를 자동 생성해야 한다.
  - `(포맷) /assets/logos/{company_name}_logo.png`
- **Fallback (대체 이미지):** 만약 해당 기업의 로고 파일이 존재하지 않거나 로드에 실패할 경우, 무조건 `/assets/logos/default_logo.png`를 노출하도록 에러 핸들링 코드를 작성한다.
